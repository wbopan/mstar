## state_bench.schemas (TaskDefinition lives here, NOT a separate module)

```python
@dataclass
class TaskDefinition:
    task_id: str
    task_summary: str
    user_id: str
    opening_message: str
    user_simulator: UserSimulatorConfig
    task_type: str | None = None
    state_requirements: list[dict[str, Any]] = []
    replay_trace_hash: str | None = None
    task_env_path: str | None = None
    task_requirements: list[dict[str, Any]] = []

    @classmethod
    def from_dict(cls, data: dict) -> TaskDefinition: ...
```

- **`task_summary` is a STRING**, not a dict. Upstream stores it as a JSON string with embedded `Task: ...\nChallenge: ...` text. The D-full renderer must NOT subscript with `["Task"]` / `["Challenge"]`.

## agents.base

```python
@dataclass(slots=True)
class AgentRuntimeContext:
    task_id: str
    user_id: str
    domain: str   # domain NAME (str), not config
    now: str
    output_dir: str | None = None
    run_idx: int | None = None
    task_summary: str | None = None
    state_requirements: list[dict] = []
    task_requirements: list[dict] = []
    config: dict = {}

class Agent(ABC):
    def __init__(self, runtime_context: AgentRuntimeContext | None = None): ...
    def prepare_conversation(self, conversation: list[Any]) -> list[Any]: ...  # hook
    def ingest_trajectory(self, trajectory: Any) -> None: ...                    # hook
    @abstractmethod
    def act(self, conversation: list[Any]) -> tuple[str, list[dict], list[Any]]: ...
```

## state_bench.client

- `class PooledLLMClient(LeastBusyPool)` — uses `AZURE_OPENAI_DEPLOYMENTS` env var; routes calls to least-busy deployment.
- We patched it with `complete_json_response` (was missing).

## openai exceptions classified as infra (retry-able / proxy-down)

`APIConnectionError`, `APITimeoutError`, `RateLimitError`, `InternalServerError`.

(Other exceptions like `ValueError`, `AuthenticationError`, `BadRequestError` are model-side issues, not infra.)

## Splits file shape

- Path: `domains/<domain>/splits/train_test_v1.json`
- **Top-level keys**: `domain`, `version`, `created_at`, `strategy`, `difficulty_source`, `splits`, `task_metadata`, `summary`
- **`splits.train`** and **`splits.test`** hold the task-id lists (NOT top-level `train` / `test`).
- Sizes: 100 train, 50 test.
- Task ID format: `<int>-<slug>` e.g. `1-return_partial_order`. Task JSON lives at `domains/<domain>/tasks/<id>.json`.

## Canonical end-to-end flow (from `scripts/run_task.py`)

```python
domain = get_domain_config(domain_name)
task = TaskDefinition.load(domains/<domain>/tasks/<task_id>.json)
env_data, _ = load_task_environment(domain, task)

agent_system_prompt = domain.agent_system_prompt.format(now=task.now, user_id=task.user_id)
env = domain.environment_class(env_data.deep_copy(), now=task.now)

runtime_ctx = AgentRuntimeContext(
    task_id=task.task_id, user_id=task.user_id, domain=domain.name, now=task.now,
    task_summary=task.task_summary,
    state_requirements=task.state_requirements,
    task_requirements=task.task_requirements,
)

agent = MstarKBAgent(client, agent_system_prompt, domain.tool_schemas, env.tool_handlers,
                    runtime_context=runtime_ctx, kb_text=retrieved_text)

trajectory = run_task(task, env_data, task.user_id, client, domain=domain, agent=agent, env=env)

# post-run scoring
trajectory.state_requirements_score = evaluate_state_requirements(task, trajectory.state_diff)
tool_calls = [tc for msg in trajectory.conversation for tc in (msg.get("tool_calls") or [])]
trajectory.task_requirements_score = task_requirements_judge.evaluate(
    task, trajectory.conversation, tool_calls, trajectory.state_diff)
trajectory.task_completion_pass = combine_task_completion(
    trajectory.state_requirements_score, trajectory.task_requirements_score)

# Final score: 1.0 if task_completion_pass == 1 else 0.0
```

## Plan corrections vs. v1 plan

1. `task_summary` is a **STRING** (e.g. `"Task: ...\nChallenge: ..."`), not a dict — D-full renderer must treat it as a string.
2. Splits are at `splits.train` / `splits.test`, not top-level `train` / `test`.
3. `load_task_environment(domain, task)` takes (domain, task), not a path.
4. `evaluate_state_requirements(task, state_diff)` — takes `task` + `StateDiff`, not `(env, requirements)`.
5. `combine_task_completion` returns `int | None` (0/1), not an object with `.passed`.
6. `Trajectory.task_completion_pass` is the bit we surface as score.
7. `domain` in `AgentRuntimeContext` is a **STRING name**, not a `DomainConfig`.
8. `tool_schemas` lives on `DomainConfig`; `tool_handlers` lives on the env wrapper.
9. We need a `TaskRequirementsJudge` instance per scorer (not just the orchestrator) — `run_task` does NOT score.
10. `resolve_task_env_path` falls back to `Path.cwd()` for relative paths — adapter must rebase against the vendor root before calling `load_task_environment`.

## Known follow-ups (out of scope for this integration)

1. **mstar `--reflect-model` Azure routing**: even with `--reflect-model azure/gpt-5.1` and `--azure-api-base http://127.0.0.1:4000/openai/`, the reflector hits "Resource not found" via litellm. Likely the reflector's `litellm.completion(...)` call doesn't honor the Azure base URL the same way the val path does.
2. **Train-side failures during offline ingestion**: `[EVAL] WARNING: 10/10 train items failed — KB partially populated`. The seed `KBProgram.write()` likely uses the toolkit-model and hits the same routing issue. Resolving (1) likely fixes this too.
3. **Phase 0 baseline**: vanilla agent on the test split × 5 runs × 3 domains for paper-grade baselines.
